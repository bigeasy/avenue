Avenue

Avenue is a file system based route generator that is used with [Locate](https://github.com/bigeasy/locate) to provide routing for the server and the browser. Avenue generates a route definition as JSON. Locate creates a routing function from the JSON. Locate is a tiny library — 0.39k minified and gzipped — that builds that one function. All the routing you need with minimal overhead, decent performance, and an intutive and expressive routing syntax. ## Installing Install Avenue from NPM. Add it to your `package.json`. ``` $ npm install avenue ``` Avenue is a server side module. It generates a route definition. The definition is used client side and server side by Locate. ## Usage Avenue runs synchronously because reading the route definitions is part of the setup of your application, just like `require`. `require` pulls in the modules you need, while the `avenue` function builds the route definition from the layout of your application folder. ```javascript var path = require('path') var avenue = require('avenue') var locate = require('locate') var definition = avenue(path.join(__dirname, 'api'), '.cgi.js') var router = locate(definition) ``` ## Diary It is tempting to have `/user-%user/profile.cgi.js` match `/user-alan/profile`, but only if it is super cheap or free. I don't see myself ever using that. I do see people abusing it: `/%product-%uuid-%color-%region`.
